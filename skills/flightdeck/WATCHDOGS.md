# Flightdeck watchdog reference

Reference doc extracted from `SKILL.md`. See [`SKILL.md`](./SKILL.md) for the load-bearing rules; this file holds detailed reference content for on-demand consultation.

## Reliability watchdogs

Five operator-facing watchdogs run inside the daemon and the `pi-agents-tmux` extension. Agents do not interact with them; they emit activity rows, wake events, and synthetic outbox payloads when child sessions misbehave.

- **agent-end** (`VSTACK_AGENT_END_WATCHDOG`, default on; grace `VSTACK_AGENT_END_WATCHDOG_GRACE_SEC`=10s) â€” if a child agent emits `agent_end` without writing a `complete_subagent` outbox within the grace window, the watchdog synthesizes a `needs_completion` outbox so the parent never silently stalls. Emits `agent.needs_completion` activity.
- **idle-stall** (`VSTACK_STALL_WATCHDOG`, default on; `VSTACK_STALL_WATCHDOG_INTERVAL_SEC`=60s, `VSTACK_STALL_WATCHDOG_THRESHOLD_SEC`=300s) â€” polls bridge-idle subagent panes whose outbox has not landed and fires a synthetic `blocked` outbox after the threshold. Emits `agent.idle_stalled`.
- **edit-loop** (`VSTACK_EDIT_LOOP_DETECTOR`, default on; `VSTACK_EDIT_LOOP_THRESHOLD_N`=5, `VSTACK_EDIT_LOOP_WINDOW_SEC`=120) â€” counts edit-tool failures inside a child agent's window; on threshold breach synthesizes a `blocked` outbox + `agent.edit_loop_blocked` activity row.
- **rate-limit** (`VSTACK_RATE_LIMIT_WATCHDOG`, default on; `VSTACK_RATE_LIMIT_MAX_ATTEMPTS`=5, `VSTACK_RATE_LIMIT_BACKOFF_LADDER`=`60,120,300,600,1800` seconds) â€” on a detected Claude API rate-limit error, schedules an exponential-backoff steer-retry. Emits activity-only `agent.rate_limit_skipped` for classifier rejections with reason `non-assistant` / `no-stopreason` / `stopreason-mismatch` / `no-prose`, `agent.rate_limit_retry` when the Pi subscriber schedules a retry, `agent.rate_limit_resolved` when a later healthy assistant turn resets that subscriber budget, `agent.rate_limit_exhausted` when attempts are spent, and `daemon.warning` if the decider is missing or fails. The pi-agents-tmux subagent wrapper also emits `agent.rate_limited` on first positive detection. The watchdog short-circuits the canonical prompt wake path while a retry is pending.
- **busy-stall** (`FD_BUSY_STALL_WATCHDOG`, default on; `FD_BUSY_STALL_THRESHOLD_SEC`=300, `FD_BUSY_STALL_CPU_PCT`=90) â€” Linux/procfs-only. Samples Pi pane process-tree CPU from `/proc`, tmux output hash, git HEAD progress, and a bounded `pi-bridge state` probe. If a Pi pane stays CPU-bound (`R` state, near a full core), bridge-unresponsive, and progress-stable beyond the threshold, the daemon emits `agent.busy_stalled` activity and wakes master with `pi-busy-stall`. Hosts without procfs cannot collect process-tree samples, so this watchdog no-ops there. Healthy long model turns that are network-bound/sleeping do not trip because they lack sustained running-state CPU burn.

All watchdogs can be hard-disabled by setting the gate env var to `0`. The canonical decision modules and parity rules live in `DEVELOPMENT.md`.
